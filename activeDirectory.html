Workgroup Vs Domain

Workgroup
No server is required
User accounts are required on every machine

Domain
A server, called a domain controller is required
User accounts are stored in active directory

Work Group

A workgroup is a peer-to-peer network using Microsoft software.
Users can use his login credentials only on his or her system and not others.
It holds an distributed administration wherein each user can manage his machine independently
A workgroup allows all participating and connected systems to access shared resources such as files, system resources and printers.


Directory Server (Domain Server)

Domain is a client/server network where user can login from any device of the office. Also known as Remote login.
It has a centralized administration and all devices can be managed from a centralized device.
It prefers a centralized storage and all the users data is stored at a centralized storage device which can be NAS or SAN.

A client-server network has clients (workstations) as well as a server (or many servers)
Clients - Computer A, Computer B, Laptop A, etc.
File Server / Directory Server - Manages user accounts and access controls

What is Active Directory?

It's a directory service available with the Windows Server platforms
It stores information in a central database and allows users to have a single user account (called a domain user account) for the network
