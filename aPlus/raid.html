RAID

Storage (Fault Tolerance)

Storage is another very important part of fault tolerance
Data loss prevention
RAID (Redundant Array of Independent Disks) - Data is copied to multiple disks

Types of RAID

Raid 0 (Striping)

Not fault tolerant
Data is not duplicated, it is "striped" (spread) across multiple disks
Advantage is speed

Raid 1 (Mirroring & Duplexing)

Is fault tolerant
Data is copied on more than one disk

Raid 5 (Striping with Parity)

Requires 3 or more disks
Data is "striped" (spread) across multiple disks along with parity
The equivalent of an entire disk is used to store parity
Example: An array of 4 disks totaling 4 terabytes, only 3 terabytes will be used for actual data storage

Raid 10

Combines RAID 1 with RAID 0
Minimum of 4 disks
Benefits from the fault tolerance of RAID 1 and the speed of RAID 0
Can only use 50% of capacity for data storage
