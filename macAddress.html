MAC Address

A MAC (Media Access Control) address is an identifier that every network device uses to uniquely identify itself on a network
No two devices anywhere in the world will have the same MAC address

00-04-5A-63-A1-66

It is made up of a 6 byte hexadecimal number that is burned into every NIC (network interface card)
The first 3 bytes identify the manufacturer of the NIC
The last 3 bytes are a unique number from the manufacturer that identifies each device on a network

The MAC address is also referred to as the physical address or the hardware address

How it is displayed

Windows - 00-04-5A-63-A1-66
Apple/Linux - 00:04:5A:63:A1:66
Cisco - 0004.5A63.A166

Purpose

The purpose of the MAC address is so network devices can communicate with each other

So what is the purpose of an IP address?
Public and private IP addresses can periodically change
MAC addresses dont change - they are permanent

A network device needs both an IP address and a MAC address
A MAC address is used to identify a device - an IP address is used to locate a device


Example

What if computer A wanted to communicate with computer B?
First computer A is going to inspect computer Bs IP address to see if it is on the same network
It knows it is on the same network since their IP is in the same group
In order to get computer B's MAC address, computer A sends our an ARP (Address Resolution Broadcast)
Once computer B tells computer A its mac address communication can finally take place


Example 2

What if computer A wants to go to google.com?
Computer A will begin by typing google.com into a web browser
The DNS will change the domain name into an IP address so that computers can understand it
Computer A will realize that the IP address is not in its local network

So the computer will forward that data to its default gateway (router)
Computer A will then send out an ARP to get the MAC address of the default gateway
Once the computer has the MAC address of the default gateway it will send its data to it

The default gateway will then inspect the IP of google.com and it will find the best path for the data to reach the destination
So it will forward the data to the next router by obtaining its MAC address with ARP
Once it reaches the final router before google.com, the router will inspect the MAC address of google.com and the data will have reached the destination


How to find your MAC address

Go into command prompt and type:
Windows - ipconfig /all
Linux / MAC - ifconfig

It can also be listed as a physical address
A computer can have multiple addresses
