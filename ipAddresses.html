Cisco CCNA | IP Addresses

Based on Layer 3 of TCP/IP - Network Layer

What is an IP Address

An IP Address is a unique identifier assigned to each device connected to a computer network (Like a digital address)
IPv4 address - 32 bit address
4 sections called octets
Each octet can contain any number between 0-255

192.168.32.152

Network section is first 3 octets | Host section is last octet
A subnet mask is used to identify the network section and the host section of the address
Subnet mask example: 255.255.255.0

Address Classes

Class A
1.0.0.0 - 126.255.255.255
Subnet: 255.0.0.0
Last 3 octets are available for host allocation
Possible Hosts: 16,777,214

Class B
128.0.0.0 - 191.255.255.255
Subnet: 255.255.0.0
Last 2 octets are available for host allocation
Possible Hosts: 65,534

Class C
192.0.0.0 - 223.255.255.255
Subnet: 255.255.255.0
Last octet is available for host allocation
Possible Hosts: 254

Small sections are carved out from each class and are called private IP addresses
Public addresses need to be unique, but private addresses can be used over and over again
Private IP addresses cannot be used over the internet
Instead of using each device as a public IP, we connect private IP's to the public ones provided to us by ISPs

Special Notes: IPv6 was invented because every address on IPv4 was allocated
