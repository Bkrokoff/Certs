Virtual Networks

We used to have server farms that would have individual computers and individual servers inside of them (100 individual servers working and connected together over the network)

With virtual servers we realized we dont need 100 separate physical servers
Instead we can create 100 virtual servers that may be located inside one device

Network Function Virtualization (NFV)

If we're replacing physical servers with virtual servers, then we're taking all of our physical network and replacing it with a virtual network, this is called network function virtualization (NFV)
We take all of our network devices and the entire network infrastructure, and move it directly into the hypervisor

All of our switching, our routing, our VLANs, our firewalls, and anything else on the network
infrastructure, are now contained within this virtual system

Provides us with the same functionality we had when we had physical switches and physical routers

Also provides additional capabilities:
When you need a new switch or a new router you simply click a few buttons inside
of the hypervisor, and you can drag and drop a brand new router or a brand new switch into your network infrastructure
You can add fault tolerance and different monitoring services, all from this network function virtualization

The Hypervisor

Virtual Machine Manager (VMM)
Manages the virtual platform and guest operating systems

Hardware Management
The hypervisor is responsible for managing access to the CPU, to memory, and to the network for all of those virtual systems

Single Console Control
One pane of glass - instead of visiting all of those individual virtual systems, you simply go to one management screen and you can control everything

VSwitch

We need to have a network that connects all of those virtual machines together
We connect that through a vSwitch, or a virtual switch

Virtual Switch
Move the physical switch into the virtual environment

Functionality is similar to physical switch
Forwarding options
Link aggregation
Port Mirroring
NetFlow

Deploy from the hypervisor
This can also be automated through the hypervisor's API so that this can be deployed and removed automatically using orchestration

Virtual Network Interface Card (vNIC)

A virtual machine needs a network interface
All of these virtual servers need a vNIC so that they can then communicate out to the rest of the network

Configured and connected through the hypervisor
Enables additional features
VLAN, aggregation, multiple interfaces
