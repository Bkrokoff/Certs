OSI Model

What is the OSI Model?

- Open Systems Interconnection Reference Model
  - It is a way to describe the way internet traffic is moving from one part of the network to another

- It's a guide (thus the term "model")
  - Don't get wrapped up in the details

- This is not the OSI Protocol Suite
  - Most of the OSI protocols didn't catch on

- There are unique protocols at every layer

- You'll refer to this model for the rest of your career
  - often

- All (Application) - Layer 7
- People (Presentation) - Layer 6
- Seem (Session) - Layer 5
- To (Transport) - Layer 4
- Need (Network) - Layer 3
- Data (Data Link) - Layer 2
- Processing (Physical) - Layer 1

Layer 1 - Physical Layer

- The physics of the network
  - Signaling, cabling, connectors
  - This layer is NOT about protocols

- "You have a physical layer problem"
  - Fix your cabling, punch-downs, etc
  - Run loopback tests, test/replace cables, swap adapter cards

Layer 2 - Data Link

- The basic network "language"
  - The foundation of communications at the data link layer

- Sometimes called the Data Link Control (DLC) protocols
  - Example of a DLC protocol: MAC(Media Access Control) address on Ethernet

- Sometimes called the "Switching" Layer

- Any communication taking place between these two NIC's (Network Interface Cards) using those MAC Addresses is referred to as a layer 2 communication

Layer 3 - Network Layer

- Sometimes called the "Routing" Layer

- Internet Protocol (IP) - Any device making forwarding decisions based on IP addresses is communicating at layer 3

- Fragments frames to traverse different networks

Layer 4 - Transport Layer

- Sometimes called the "Post Office" Layer
  - Parcels and Letters
  - Describes how data is being delivered and where data is being delivered into a system

- TCP (Transmission Control Protocol) and UDP (User Diagram Protocol)

- This layer is used when you are accessing a webpage, but the website is so large that data cannot be sent across the network in a single frame
- Instead it is split up into separate frames and sent across the network and then put back together when it reaches the other side

Layer 5 - Session Layer

- Communication management between devices
  - Start, Stop, Restart

- Control protocols, tunneling protocols used to begin the communication of data between one device and another

Layer 6 - Presentation Layer

- Character Encoding

- Application Encryption

- Takes the data and puts it into a form that we can understand

- Often combined with application layer
  - The functionality is closely associated with our ability to use these applications

Layer 7 - Application Layer

- The layer we see

- HTTP, FTP, DNS, POP3

Real World Example

- Application - HTTP, FTP, SMTP
- Presentation - Application Encryption (SSL/TLS)
- Session - Controls protocols, tunneling protocols, Authentication/Authorization
- Transport - TCP segment, UDP segment
- Network - IP Address, Router, Packet
- Data Link - Frame, MAC address, Extended Unique Identifier (EUI-48, EUI-64), Switch
- Physical - Cables, fiber, the signal itself, hub, network cabling, and repeater

- Example with wireshark

PDU Units

- Application - Data
- Presentation - Data
- Session - Data
- Transport - Segment
- Network - Packet

- Physical - Bit

Real World Example 2

Sending Steps

Step 1. Application Layer - Gets ready to send email using SMTP (Simple Mail Transfer Protocol)
Step 2. Presentation Layer - Data is formatted in a way the receiving device will understand (ASCII)
Step 3. Session Layer - Session for receiving mail server is started
Step 4. Transport - Decides to use TCP or UDP to make sure every packet is delivered | Source and destination port added to data
Step 5. Network - IP address of the mail server is added as the destination | Source IP address is also added to the data
Step 6. Data - The MAC address of the router and the source MAC address of the host is added to the data
Step 7. Physical - The data is sent out on the network using ethernet

Receiving Steps

Steps from above are done in reverse, starting from the bottom up

Troubleshooting

Layer 1 (Physical) - Are all the cables plugged in | Is the network card functioning | Faulty Cable
Layer 2 (Data) - Maybe a switch has gone bad
Layer 3 (Network) - Is the router functioning | Do I have the right IP address
Process continues from there...
