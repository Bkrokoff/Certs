Real World Example

Sending Steps

Step 1. Application Layer - Gets ready to send email using SMTP (Simple Mail Transfer Protocol)
Step 2. Presentation Layer - Data is formatted in a way the receiving device will understand (ASCII)
Step 3. Session Layer - Session for receiving mail server is started
Step 4. Transport - Decides to use TCP or UDP to make sure every packet is delivered | Source and destination port added to data
Step 5. Network - IP address of the mail server is added as the destination | Source IP address is also added to the data
Step 6. Data - The MAC address of the router and the source MAC address of the host is added to the data
Step 7. Physical - The data is sent out on the network using ethernet

Receiving Steps

Steps from above are done in reverse, starting from the bottom up

Troubleshooting

Layer 1 (Physical) - Are all the cables plugged in | Is the network card functioning | Faulty Cable
Layer 2 (Data) - Maybe a switch has gone bad
Layer 3 (Network) - Is the router functioning | Do I have the right IP address
Process continues from there...

What is the OSI Model?

- Open Systems Interconnection Reference Model
  - It is a way to describe the way internet traffic is moving from one part of the network to another

- It's a guide (thus the term "model")
  - Don't get wrapped up in the details

- This is not the OSI Protocol Suite
  - Most of the OSI protocols didn't catch on

- There are unique protocols at every layer

- You'll refer to this model for the rest of your career
  - often

- All (Application) - Layer 7
- People (Presentation) - Layer 6
- Seem (Session) - Layer 5
- To (Transport) - Layer 4
- Need (Network) - Layer 3
- Data (Data Link) - Layer 2
- Processing (Physical) - Layer 1

Layer 1 - Physical Layer

- The physics of the network
  - Signaling, cabling, connectors
  - This layer is NOT about protocols

- "You have a physical layer problem"
  - Fix your cabling, punch-downs, etc
  - Run loopback tests, test/replace cables, swap adapter cards
