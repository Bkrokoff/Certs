Network Types

Network types depend on how large they are and how much of an area they cover geographically

Peer-to-peer Network

With a peer to peer network, there is not a server or a client
Instead, every device is a server and a client - everyone talks to everyone

Advantages
This is a very easy way to deploy and application
Low cost to implement

Disadvantages
Difficult to administer this type of system
Difficult to secure since application and authentication process is distributed

Client Server

With client-server communication the responsibilities are split
You have individual clients and then you have a centralized server, so when clients need to access data everyone will access the same server

Clients are not communicating with each other directly or through the server
All clients talk to the server, then the server talks to the clients

Advantages
This gives you very good performance because you have direct connection to the server that you can manage and maintain
Allows for easy administration because you can configure application and authentication at the central point where the server is

Disadvantages
Cost and complexity
There is additional hardware and software that you'll need - Server + a place to store/maintain it
A bit complex since you have a centralized server that needs to be administered

PAN (Personal Area Network)

Network that is used on a personal level

It is a small network that is used for connecting devices such as smartphones, tablets, and laptops
Connect to each other using wireless technologies: bluetooth, infrared, NFC (near field communication)
Can also be connected using a wired connection: USB

Generally used for transferring small files: music, photos, calendar appointments, etc

LAN (Local Area Network)

A group of devices (computers, servers, switches, and printers) that are located in the same building in close proximity to each other

The most common type of LAN is Ethernet LAN, where two or more computers are connected to ethernet cables using a switch
Also 802.11 wireless connection

WLAN (Wireless Local Area Network)

Uses wireless communication instead of wired communication
A WLAN is a LAN but without using cables (802.11 technologies)

WLAN will typically have a router or a wireless access point for wireless devices to communicate

You can expand coverage with additional access points to span a larger area

CAN (Campus Area Network)

A CAN is an network that joins two or more LANs together within a limited area

For example, a CAN could be a university that has multiple buildings in the same general area that are connected to each other to form a larger network

LAn technologies: fiber connected, high speed ethernet
Fiber is often run in the ground without a third party provider

MAN (Metropolitan Area Network)

Larger than a CAN and spans over several buildings in a city or town

MANs are typically connected using high speed connection such as fiber optic cable

It is a high speed network that gives the ability for sharing data and resources within a city, often used by governments

SAN (Storage Area Network)

A special high-speed network that stores and provides access to large amounts of data

A dedicated network that is used for data storage

Consists of multiple disk arrays, switches, and servers

SANs are not affected by network traffic since SANs aren't a part of a LAN, it is partitioned off

NAS (Network Attached Storage)

One of the main reasons for adding connections on local and wide area networks is so user can access their data that is in a central location that is stored on something like a NAS

NAS contains multiple drives that is able to store large amounts of data in a single area
NAS can usually access files at the file level - file level access means that any changes made to the file have to be done to the entire files that's located on that device

SAN allows you to make block-level changes so it can be done very quickly and very efficiently compared to a NAS

WAN (Wide Area Network)

Largest type of network consisting of: LANs, CANs, and MANs

It is a network that spans over a large geographical area such as a country, continent, or even the globe

The internet is an example of a WAN

MPLS (Multiprotocol Label Switching)

This is communication through the WAN, but it uses labels to be able to determine how we route and forward that traffic through the WAN

Advantages
We can use almost any type of connection and we can put almost any type of data inside an MPLS network
This means that we can send IP traffic, ethernet frames, or any other type of data we'd like through an MPLS network

This makes MPLS relatively easy to get up running and have all our data transferred between two locations

With MPLS we make forwarding decisions based on the label switching

If we have data on one side of the network and we would like to get it to the other side of the MPLS network, we send that data into the first provider edge router that closes to us
The edge router will insert a label into the data (pushing)
Then it will know how to forward that traffic through the internals of the provider switch network
When the data reaches the other side, the label is popped off and the remaining data is sent to the customer edge router

mGRE (Multipoint Generic Router Encapsulation)

With MPLS we have to create an initial configuration that defines where all the different sites may be located and what labels may be used to switch data to those locations

Multipoint Generic Router Encapsulation

Used extensively for Dynamic Multipoint VPN (DMVPN)
Common on cisco routers

What's nice about these dynamic connections is they are only created when they're needed
Torn down when you don't have any other information you need to send

If there is a break in the connection - your VPN can rebuild itself
Remote sites communicate to each other

All of this is considered to be a dynamic mesh, since you can send data to whatever site you would like based on where the data needs to go

More efficient network and only using connectivity when you have to have it

SD-WAN (Software Defined Networking in a Wide SArea Network)

The SD stands for software-defined which means that our wide area network can be defined however we would like to lay it out

Especially useful if we have cloud-based applications, and you need the network to understand where data needs to go at any particular time

Allows for communicating directly to the cloud instead of going through one central data center

Software defined aspect of SDWAN determines how data is routed and sent based on types of services that are required
Makes for more efficient network
