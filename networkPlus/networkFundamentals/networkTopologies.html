Network Topologies

What is a network topology?

A topology is the layout of how a network communicates with different devices

There are a couple of different categories of topologies
Wired and wireless

Wired Topologies

Star Topology (Hub and Spoke)

Most common wired topology
All computers are connected to a central wiring point, such as a hub or a switch

All data on a star network passes through the central point before continuing to its destination

One of the major benefits of this topology is that if one of the computers failed or if there was a break in the cable, other computers would not be affected because each computer has its own cable connection

A disadvantage is that if the central hub or switch fails, then all the computers on that central point would be affected (Single point of failure)
If this happens the entire network goes down

Used in most large and small networks

Ring Topology

Type of network configuration where each computer is connected to each other in the shape of a closed loop or ring

Every computer in the ring has exactly two neighbors for communication purposes

Each data packet is sent around the ring until it reaches its final destination

This kind of topology is rarely used today

The advantages of a ring topology are that it is easy to install and easy to troubleshoot

However, the disadvantage would be, that if just one of these computers goes down or if there was a single break in the cable, then all data flow would be disrupted

Not used in LANs, but still used in many Metro Area Networks (MANs) and Wide Area Networks (Wans)

It works in these environments because once devices realize that data may not be able to traverse the connection, it will loopback around and travel the other direction on the ring

Bus Topology

The kind of network setup where each of the computers and network devices are connected to a single cable or backbone
This backbone is a coaxial cable

They connect to the cable using special connectors called BNC, also known as T connectors
Advantages - It is fairly cheap and easy to implement

Disadvantage - Requires that the cable is terminated at both ends using terminators

In order for this setup to remain operational, there must not be any open connection, including the ends that attach to the computers
If a computer is removed or if the terminators are loose/missing, then the cable would be open and data would bounce back
This bounce is known as signal reflection, if this happens data flow would be disrupted

Mesh Topology

Each computer on the network is connected to every other computer on the network

By having so many connections it handles failure very well

In the illustration there are 4 computers with 3 connections on each computer, so there is 12 connections total

Advantage - High redundancy level. If one or more connections fail, the computers would still be able to communicate with each other

Because of the amount of cabling and network cards that have to be used, mesh topologies can be expensive so they are rarely used on local area networks (LANs)

Mainly used on wide area networks, like the internet
Since the internet is made up of numerous routers all over the world that are connected to each other, if a few routers go down data will still be able to transfer through rerouting
Internet is very redundant since it uses a mesh topology

Wireless Topologies

Infrastructure Topology

This topology uses a combination of wired and wireless devices

It is similar to the star topology, where you have wired devices physically connected to the switch, and a wireless access point that's connected by a cable to the same switch

Wireless access point acts like a bridge between the wireless network and the wired network

Infrastructure topology can have multiple wireless access points depending on the needs of the network

Ad Hoc Topology

Ad Hoc does not rely on any infrastructure, such as cables, routers, servers or wireless access points

All devices in an ad hoc network wirelessly connect to other devices in a simple peer to peer network

Devices connect directly to each other without a centralized device, such as a wifi router or access point
Because of this, each device is responsible for its own security and permissions

Ad hocs are useful for setting up a quick wireless network on the fly, where devices can share data without the need of an existing wireless network

Wireless Mesh Topology

Similar to wired mesh topologies, where devices are interconnected with each other, except here they are wirelessly connected

In a wireless mesh, each wireless access point will talk to other wireless access points to create a seamless internet connection for wireless devices to connect to

Device will connect to nearest wireless access point

A wireless mesh topology is very redundant because the internet connections is spread out over many wireless access points

Hybrid Topology

A combination of one or more physical topologies
Most networks are a hybrid
