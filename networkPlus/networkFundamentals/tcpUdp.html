TCP vs UDP

TCP

The communication between two computers needs to be “good” and reliable, to guarantee the data is received correctly

TCP (Transmission Control Protocol) helps ensure no data is missing or out of order

TCP is a connection-oriented protocol, which means that it must first acknowledge a session between the two computers that are communicating
It verifies this by using a three-way handshake

First Step - Computer will send a message called SYN
Second step - The receiving computer will send back an acknowledgement message, telling the sender it has received the message (SYN ACK)
(The SYN ACK message's purpose is not only to acknowledge the client's SYN. This is only the ACK part. The SYN part is the server's own SYN, which is actually what the client ACKnowledges via the final ACK)
Third Step - The sending computer sends another acknowledgement message back to the receiver (ACK RECEIVED)

Once this has taken place, the data can be delivered

TCP doesn't guarantee that data arrives at the receiver. It detects when a segment hasn't arrived and resends it. The resent data may also be lost. After a certain number of failed retransmissions of a segment, TCP will give up and abort the connection

UDP

UDP (User Datagram Protocol) is connection-less, which means it does not establish a connection and it does not guarantee data delivery

Known as the “fire-and-forget” protocol because it sends the data, but it does not care if the data is received at the other end

Since there is less overhead involved in not guaranteeing data delivery UDP is faster than TCP
